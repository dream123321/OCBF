字段Field默认值Default必填Required说明Description
parameter.mlp_encode_modeltrue否No默认启用 OCBF 编码采样。Encoded sampling is enabled by default.
parameter.encoding_cores2否No采样专用 descriptor 编码并行核数。Sampling-side descriptor encoding worker count.
parameter.iw_methodFreedman_Diaconis否No区间宽度方法，支持 `Freedman_Diaconis`、`self_input`、`scott`、`std`。Interval-width method. Supported values include `Freedman_Diaconis`, `self_input`, `scott`, and `std`.
parameter.iw0.01否No显式宽度值。Explicit interval width value.
parameter.iw_scale1.0否No宽度缩放系数。Interval-width scaling factor.
parameter.body_list["two", "three"]否No覆盖统计的 body 项。Body terms used in coverage statistics.
parameter.mtp_typel2k2否NoMTP 模板类型。MTP template type.
parameter.selection_budget_schedule[20, 15, 10]否No各阶段结构预算。Structure budget per stage.
parameter.coverage_threshold_schedule[99.5, 99.9, 99.95]否No各阶段覆盖率阈值。既支持标量列表，也支持逐元素阈值列表；每个内层列表长度必须是 `1` 或等于元素数。Coverage thresholds per stage. It supports either a scalar list or per-element threshold lists. Each inner list length must be `1` or equal to the number of elements.
parameter.coverage_rate_methodmean否No覆盖率聚合方法。Coverage aggregation method.
parameter.coverage_calculation_modeper_configuration否No覆盖率口径：`global` 或 `per_configuration`。Coverage mode: `global` or `per_configuration`.
parameter.report_per_configuration_detailstrue否No是否输出每个源配置的覆盖率和选中数量。Whether to print per-source coverage and selected counts.
OCBF 覆盖率口径：`global` 表示把所有候选结构合并后统一计算一次覆盖率；`per_configuration` 表示按每个源结构系统 / 每个源配置分别计算覆盖率，再做保守聚合。若 `report_per_configuration_details=true`，日志会打印每个源配置的覆盖率和最终选中数。OCBF coverage interpretation: `global` computes coverage once after all candidate structures are merged together. `per_configuration` computes coverage separately for each source configuration / source structure system and then aggregates the result conservatively. If `report_per_configuration_details=true`, logs will also print per-source coverage and selected counts.
`coverage_threshold_schedule` 写法：最简单的写法是标量列表，例如 `[99.5, 99.9, 99.95]`，表示每一阶段对所有元素使用同一个覆盖率阈值。如果体系有多个元素，也可以写成逐元素列表。例如一个二元体系可以写成 `[[99.5, 99.2], [99.9, 99.6], [99.95, 99.8]]`，表示第一阶段分别对元素 1 和元素 2 使用不同阈值。`coverage_threshold_schedule` formats: The simplest form is a scalar list such as `[99.5, 99.9, 99.95]`, which applies the same threshold to all elements at each stage. For multi-element systems, you can also provide per-element lists. For example, a two-element system can use `[[99.5, 99.2], [99.9, 99.6], [99.95, 99.8]]`, meaning each stage applies different thresholds to element 1 and element 2.

To use a custom potential, set `use_universal_potential=false`, provide `reduce.mtp_path`, provide `parameter.ele`, and make sure `parameter.mtp_type` matches the potential type.
